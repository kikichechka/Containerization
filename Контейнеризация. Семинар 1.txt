sudo unshare --net bash = пространство имен становится постоянным, создается привязка монтирования в файл

sudo unshare --pid bash = создание нового пространство имен PID. Если файл указан, то пространство имен становится постоянным, создавая привязку монтирования в файл. Создание постоянного пространства имен PID завершится ошибкой, так как опция --fork не указана.

sudo unshare --pid --fork bash = создание нового пространство имен PID. Если файл указан, то пространство имен становится постоянным, создавая привязку монтирования в файл. 

sudo unshare --fork bash = разветвление указанной программы как дочерний процесс unshare. Это удобно при создании нового пространства имен PID.

Mkdir – создание новой директории
Ls – посмотреть файлы и папки в текущей директории
Cd – сменить директорию
Cp file1 file2 – скопировать file1 как file2
Ldd – список зависимостей
Chroot – смена корня
Ip netns list – посмотреть сетевые пространства имён

Ip netns exec namespace_name bash – запустить командную оболочку с  изоляцией сетевых интерфейсов


Ip link set dev lo up – поднять сетевой интерфейс
Unshare [--pid|--net|--fork|--mount-proc] /bin/bash – изоляция процесса
